# UnMeet 产品定义 v2.0

> 2026年7月 | 基于竞争分析重定位

---

## 一、一句话定义

**自动追踪所有会议平台 → 每周生成一份老板看得懂的成本 & 低效报告。**

---

## 二、核心洞察

### 2.1 会议工具市场已被大厂占领

飞书妙记、腾讯 AI 小助手、Teams Copilot、Zoom AI Companion 已经覆盖了会议纪要、转录、AI 摘要。**任何第三方工具在这一层与大厂竞争都是自杀。**

### 2.2 大厂故意留出的空白

飞书不会告诉你"用飞书开会花太多钱了"。  
Teams 不会建议你"少用 Teams 开会"。  
Zoom 不会说"这个会不值得开"。

**会议平台是利益冲突方——他们靠你多开会赚钱。UnMeet 是唯一没有利益冲突的第三方。**

### 2.3 付钱的人不是开会的人

开会的人痛恨会议但不会主动买工具。**付钱的人是老板/CFO——他们要看 ROI。** 所以产品形态不应该是"开会辅助工具"，而应该是"会议成本治理报告"。

## 三、产品形态

```
┌─────────────────────────────────────────────┐
│                                             │
│   UnMeet Desktop App（系统托盘，静默运行）      │
│                                             │
│   用户安装后 → 忘记它 → 它会：                │
│   1. 自动检测所有平台的会议                    │
│   2. 默默记录时长、时间、平台                  │
│   3. 每周自动生成一份报告                      │
│   4. 发给老板（或自己看）                      │
│                                             │
│   不需要任何手动操作。                         │
│   不需要录音。不需要日历。                     │
│                                             │
└─────────────────────────────────────────────┘
```

### 3.1 三个界面

| 界面 | 给谁 | 功能 |
|------|------|------|
| **系统托盘** | 所有人 | 静默运行，开会时图标变绿。右键：本周统计 + 退出 |
| **Dashboard** | 想看数据的人 | 今日/本周/历史会议列表 + 低效标记。Rating 可选 |
| **周报** | 老板/自己 | PDF/邮件。团队总成本、低效会议清单、省钱建议 |

### 3.2 不做的事

- ❌ 不录音
- ❌ 不做会议纪要
- ❌ 不做 AI 转写
- ❌ 不做参与度分析
- ❌ 不做日历集成（可选但非核心）

## 四、核心功能

### 4.1 会议检测引擎

| 功能 | 说明 |
|------|------|
| 系统级窗口检测 | Windows：PowerShell 读前台窗口标题。Mac：AppleScript |
| 平台覆盖 | Zoom / Teams / Google Meet / 腾讯会议 / VooV / 飞书 / Webex / RingCentral / Whereby / Slack Call / Discord |
| 检测逻辑 | 窗口标题匹配 → 开始计时。切换窗口暂离有 6 秒 grace period |
| 自动评级 | 会议结束后通知：1-5⭐ 评分（可选） |

### 4.2 低效检测引擎（核心竞争力）

基于**零额外数据采集**的三个维度：

| 维度 | 信号 | 阈值 |
|------|------|------|
| **时间异常** | 超长会 | > 90 分钟 |
|  | 碎片化 | ≥ 5 个短会/天，均 < 15min |
|  | 背靠背 | 间隔 < 5 分钟 |
|  | 黄金时段被占 | 上午 9-11 点 |
|  | 非工作时间 | 晚 8 点后 / 周末 |
| **重复陷阱** | 惯性例会 | 同一时段重复 ≥ 4 周 |
|  | 时长膨胀 | 近 4 周比前 4 周长 > 30% |
|  | 评分走低 | 连续 3 周均分 < 3 |
| **成本异常** | 高价低效 | 成本 > ¥500 且无评分 |
|  | 累计黑洞 | 单一例会月成本 > 总量 15% |

每项扣分，总分 0-100：
- ≥70 🟢 高效
- 50-69 🟡 存疑
- 30-49 🟠 低效
- <30 🔴 严重低效

### 4.3 周报生成

每周自动生成，包含：

```
UnMeet 周报 — 2026年7月21日-27日

本周团队会议总结
  总时数: 87 小时（人均 12.4 小时）
  总成本: ¥15,660（按人均 ¥180/h 估算）
  会议数: 42 次
  低效会议: 3 次（7%）

需要重新评估的 3 个会议:

🔴 周一下午产品评审（第 12 周）
   90 分钟 · ¥1,200 · 连续 4 周无评分
   → 建议：提前发材料，会控在 30 分钟内只做决策

🟠 周三上午全员站会
   35 分钟（超时 50%）· 每天重复 · 占用 9:15 黄金时段
   → 建议：改为 Slack 异步，保留周二/周四语音

🟡 周五下午 1:1
   背靠背接在 2 个会后 · 无缓冲
   → 建议：移到周四上午

预计月度节省:
  12 小时团队时间 ≈ ¥8,400/月
  如果全年执行 ≈ ¥100,800/年
```

### 4.4 团队版

- **Workspace**：管理员创建，成员加入
- **成员管理**：姓名 + 时薪（用于成本计算）
- **团队周报**：聚合所有人的数据
- **导出**：JSON / PDF
- **隐私**：Admin 只看聚合数据（团队总时长/成本），看不到个人具体会议

## 五、与竞品的本质区别

| | 飞书/Teams/腾讯会议 | UnMeet |
|------|------|------|
| 定位 | 帮你开会 | 帮你少开会 |
| 利益 | 多开会 = 多用平台 | 少开会 = 用户省钱 |
| 核心功能 | 纪要、转录、录制 | 成本、低效检测、周报 |
| 跨平台 | ❌ | ✅ |
| 数据 | 云端 | 本地优先 |
| 目标 | 开好每一个会 | 砍掉不值得的会 |

**UnMeet 不跟飞书比功能，它跟飞书比立场。**

## 七、技术栈

| 层 | 选型 |
|------|------|
| 桌面客户端 | Electron + Node.js |
| 窗口检测 | Windows PowerShell / Mac AppleScript |
| 数据存储 | 本地 JSON（AppData） |
| 低效引擎 | 纯逻辑，不需要 AI API |
| 周报生成 | HTML → PDF（Electron 内置 printToPDF） |
| 团队同步（未来） | Supabase（聚合数据上传，不含会议内容） |

| calendar.js 中的 AI 纪要 | 大厂秒杀 |
| 参与度分析 | 依赖录音 |
| DeepSeek 转录 | 不再需要 |

## 八、Go-to-Market

### 8.1 目标用户

- **主用户**：中小公司老板 / 部门负责人（20-200 人）
- **次用户**：自由职业者、远程团队 lead
- **使用场景**：用飞书/企业微信/Teams 但想管理会议成本的团队

### 8.2 病毒钩子

每周报告里的省钱数字 → "我们的会议成本降低了 40%" → 老板在朋友圈/LinkedIn 晒 → 自然获客。

### 8.3 定价锚

"一顿饭钱帮公司每月省 ¥8,400。" （老板心理：¥58/月换 ¥8,400，闭眼买。）
